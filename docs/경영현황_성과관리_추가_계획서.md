# 경영 현황 — 성과 관리(Performance) 뷰 추가 계획서

> 작성일: 2026-07-03 · 참고: 사용자 제공 "사업 성과관리 시스템" 대시보드 이미지
> 관련: `경영자_대시보드_개발_계획서.md`(P1·P4 완료), 메모리 `exec-dashboard`

---

## 1. 배경

경영 현황(`/exec`)은 현재 **전사 요약(KPI·부문평가·경영알림)** 중심이다. 사용자가 제공한 참고 화면("사업 성과관리")은 여기에 없는 **성과 분석형 시각화**를 담고 있어, 경영 현황에 **성과 관리 뷰**로 추가한다.

### 참고 화면 구성 분석
| 영역 | 참고 화면 요소 | 성격 |
|---|---|---|
| 상단 KPI 4 | 모니터링 대상 156개 · 전년比 성장률 132% · 전월比 -28% · 잔여기간 D-235 | 성장률·카운트·D-day |
| 목표 대비 성과 | **반원 게이지 93%** | 목표 달성 게이지 |
| 대조군 대비 성장 추이 | **라인 2계열**(대상 vs 대조군) | 벤치마크 비교 |
| 대상 특성 | **도넛 2개**(업종·지역) | 구성 비중 |
| 전년 대비 매출 추이(최근 10주) | **+/- 발산 막대**(주간 증감) | 증감 파형 |
| 대상 상세 정보 | 표(사업장·지역·전/후 매출·증감) | 드릴 상세 |

---

## 2. 보유 자산 매핑 (재사용 우선)

대부분 **이미 있는 차트로 구현 가능**하다. 신규는 발산 막대 하나뿐.

| 참고 요소 | 우리 컴포넌트 | 신규 여부 |
|---|---|---|
| KPI 4타일 | 기존 exec 타일 패턴(Card+Sparkline) | 재사용 |
| 목표 대비 성과 게이지 | `@/shared/ui/charts/Gauge`(value/max/label/color) | 재사용 |
| 대조군 대비 추이 | `LineChart`(series 2개) | 재사용 |
| 특성 도넛 2개 | `Donut`(data/centerTop/centerSub) | 재사용 |
| 대상 상세표 | `RTable` 또는 인라인 table + 증감 화살표 | 재사용 |
| **주간 매출 증감(+/-)** | **신규 `DivergingBars`**(0 기준선 위=상승/아래=하락) | ⚠ 신규 1종 |

> `DivergingBars`: `GroupedBars`(동일 축 그룹)로는 +/- 발산을 표현 못 함 → 0 기준선 기준 위(빨강)/아래(파랑) 막대 + 값 라벨의 **작은 SVG 컴포넌트**를 `src/shared/ui/charts/`에 신설.

## 3. 도메인 매핑 (핵심 결정 필요)

참고 화면은 **여러 사업장 vs 대조군**의 매출 성장을 모니터링한다. MES 맥락에서 "사업장/대조군"을 무엇에 대응시킬지 정해야 한다(§7-1). 데모는 샘플이므로 어느 쪽이든 화면은 동일하게 구현 가능.

| 참고 개념 | MES 대응 후보(택1) |
|---|---|
| 모니터링 대상(사업장) | ⓐ 생산 라인 · ⓑ 공장/사이트 · ⓒ 주요 거래처(고객사) · ⓓ 제품군 |
| 대조군(벤치마크) | 전년 동기 · 목표치 · 타 라인 평균 |
| 성장률/매출 | 생산량·매출·수율 성장률 |
| 목표 대비 성과(게이지) | 목표 달성률(부문 점수와 연계 가능) |
| 업종·지역 특성 도넛 | 제품군 구성 · 라인/지역 구성 · 거래처 업종/지역 |
| 대상 상세표 | 라인/거래처별 전기 대비 매출·증감 |

**권장 기본안(데모)**: 대상 = **주요 거래처/제품군**, 대조군 = **전년 동기**, 지표 = 매출·성장률. 참고 화면과 의미가 가장 자연스럽게 대응(매출 성장 모니터링)한다.

---

## 4. 배치 방식 (경영 현황 내)

세 가지 안. **권장 = B(별도 서브 화면)**.

| 안 | 내용 | 장단 |
|---|---|---|
| A | `/exec` 랜딩 하단에 섹션으로 이어 붙임 | 한 화면에 다 보이나 스크롤 길고 무거움 |
| **B(권장)** | 경영 현황 그룹에 **서브 화면 `/exec/performance`(성과 관리)** 신설 | 랜딩은 요약 유지, 성과 분석은 전용 화면. 메뉴 자연 확장 |
| C | 랜딩 상단 탭(종합|성과관리) 세그먼트 | 전환 빠르나 셸 탭 구조 변경 필요 |

- B 채택 시: `menu-tree`의 `M_EXEC > G_EXEC_DASH`에 `S_EXEC_PERF`(성과 관리, `/exec/performance`) 추가. 랜딩 KPI/알림에서 "성과 관리 보기" 진입 링크 제공.

## 5. 데이터 설계 (단일 소스 격리 유지)

기존 경영 대시보드와 동일 원칙([[data-layer-pattern]]). 화면은 **단일 소스 함수 하나만** 호출.

- `src/modules/exec/performance.mock.ts`: `getExecPerformanceData()`(현재 샘플, 후속 실집계 교체 지점).
- 반환 구조(초안):
  ```ts
  interface ExecPerformanceData {
    kpis: PerfKpi[];              // 4타일(대상수·전년比·전월比·D-day)
    goalGauge: { value: number; label: string };   // 목표 대비 성과
    trend: { labels: string[]; target: number[]; control: number[] };  // 대조군 비교
    byCategory: DonutDatum[];     // 업종/제품군 특성
    byRegion: DonutDatum[];       // 지역/라인 특성
    weekly: { label: string; delta: number }[];    // 주간 매출 증감(+/-)
    rows: PerfRow[];              // 대상 상세표(명/지역/전·후 매출/증감)
  }
  ```
- 실집계 전환(후속 P5)은 이 함수 내부만 `services/reporting`로 교체 → 화면 무변경.

## 7. 확인 필요 (착수 전)

1. **도메인 매핑**: 모니터링 "대상/대조군"을 무엇으로? (§3 — 권장: 대상=거래처/제품군, 대조군=전년 동기). MES 실데이터 성격상 "매출"이 맞는지, 아니면 "생산량/수율"로 볼지.
2. **배치**: 별도 서브 화면 `/exec/performance`(권장) vs 랜딩에 섹션 추가.
3. **지표 정의**: 성장률 산식·목표치·D-day 기준(사업 기간). 데모는 샘플로 시작.
4. **특성 도넛 축**: 업종/지역(참고 그대로) vs 제품군/라인(MES 친화).

## 8. 요약

- 참고 "사업 성과관리" 화면을 경영 현황에 **성과 관리 뷰**로 추가.
- **신규는 발산 막대(DivergingBars) 하나뿐**, 나머지(게이지·도넛·라인·표)는 보유 컴포넌트 재사용.
- 권장: 경영 현황 그룹에 **서브 화면 `/exec/performance`** 신설, 단일 소스 `getExecPerformanceData()`로 샘플 구현(실집계는 후속).
- 착수 전 §7의 도메인 매핑(대상/대조군)·배치만 확정하면 바로 진행 가능.
