# 전결 규정에 따른 결재선 정립 및 판정 로직

제공해 주신 **전결 규정 (PDF)**의 모든 행을 분석하고, **"각 행의 첫 번째 동그라미(O)가 위치한 열이 곧 기안자(Drafter)"**라는 원칙을 적용하여 동적 결재선 및 기안 권한 판정 로직을 정립합니다.

---

## 1. 전결 구분표의 해석 규칙 (수정본)

### 1.1 열(Column)의 의미와 역할
- **기안자(Drafter) 판정**: 각 행(업무 항목)에서 **가장 왼쪽에 위치한 첫 번째 O**가 위치한 열의 직책이 해당 업무의 **기안자**가 됩니다.
  - 담당(O)로 시작 $\rightarrow$ **팀원** 기안
  - 팀장(O)로 시작 $\rightarrow$ **팀장** 기안
  - 본부장(O)로 시작 $\rightarrow$ **본부장** 기안
- **결재선 구성**: 첫 번째 O의 우측에 위치한 O들은 **순차적인 결재/검토자**가 되며, 가장 우측의 O가 최종 **전결권자**가 됩니다.
- **참조**: '참조' 열에 적힌 직책은 결재가 완료된 후 문서를 공유받습니다.

---

## 2. 실제 전결 규정 PDF 기반 결재선 매핑 규칙

PDF 상의 모든 주요 업무 항목에 대해 기안자 및 결재선을 아래와 같이 정의합니다.

### 2.1 인사 (휴가 / 외근 / 연장근로)

| 업무 항목 | 시작 O (기안자) | 이후 O 및 참조 (결재선) | 해석 및 결재선 흐름 |
| :--- | :---: | :---: | :--- |
| **1) 휴가 - 가. 팀원** | 담당 | 팀장 | **팀원 기안** $\rightarrow$ 팀장 결재 (전결) |
| **1) 휴가 - 나. 팀장** | 팀장 | 본부장 (결재), 대표이사 (참조) | **팀장 기안** $\rightarrow$ 본부장 결재 (전결) $\rightarrow$ 대표이사 참조 |
| **1) 휴가 - 마. 임원** | 본부장 | 대표이사 (참조) | **본부장 기안** $\rightarrow$ 대표이사 참조 (전결 없이 보고 참조) |
| **2) 연장근로 - 가. 휴일근무** | 담당 | 팀장, 본부장, 대표이사 (참조) | **팀원 기안** $\rightarrow$ 팀장 검토 $\rightarrow$ 본부장 결재 (전결) $\rightarrow$ 대표이사 참조 |
| **3) 외근 - 가. 팀원** | 담당 | 팀장 (참조) | **팀원 기안** $\rightarrow$ 팀장 참조 (자가전결 후 팀장에게 통보) |
| **3) 외근 - 나. 팀장** | 팀장 | 본부장 (참조) | **팀장 기안** $\rightarrow$ 본부장 참조 |

## 3. 동적 결재선 룰 엔진 반영 및 검증 전략 (기능 구현 방식)

### 3.1 기안 권한 차단 메커니즘
- 사용자가 전자결재 작성 화면에서 특정 서식(예: 국내출장신청서)과 금액을 입력할 때, 해당 서식의 금액 규칙을 확인합니다.
- 만약 현재 로그인한 사용자의 직책이 **규정에 명시된 기안자 직책(시작 O)보다 하위 직책**이라면, 상신(제출) 버튼이 비활성화되거나 경고창("이 금액 대의 기안 권한이 없습니다. 팀장 이상만 기안 가능합니다.")을 띄워 작성 단계에서 제어합니다.

### 3.2 상위 결재라인 동적 단축 (Skimming)
- 만약 기안 조건이 "담당(O) 시작"이어서 팀원이 기안했으나, 실제 기안자가 "팀장"인 경우:
  - 기안자가 이미 팀장이므로 자신의 직책 이하의 결재 단계(팀장 검토)는 자동으로 생략되고, 다음 단계인 **본부장 결재**부터 시작됩니다. (중복 결재 방지)
