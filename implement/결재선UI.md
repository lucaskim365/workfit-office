# 전자결재 결재선 UI/UX 개선 계획 및 참조자 표현 방안 보고서

본 보고서는 결재 상신 및 문서 상세 조회 시 결재선 정보가 텍스트 중심의 수평형 리스트로 표현되어 직관성이 다소 떨어지는 점을 보완하고, 시각적으로 흐름이 한눈에 보이는 **정사각형 카드 형태의 수평 결재선 UI**로 개편하기 위한 설계 계획 및 **참조자 표현 기법**에 관한 연구 보고서입니다.

---

## 1. 현황 및 개편 필요성
* **현재의 UI 상태:** 텍스트와 단순 나열 형식(수평/수직 리스트) 위주로 되어 있어 결재권자들의 단계적 순서와 승인 경로가 직관적으로 부각되지 않음.
* **개선 목표:** 결재 단계의 진행 흐름(1단계 ➔ 2단계 ➔ 3단계)을 **정사각형 박스 카드**와 **이동 화살표(➔)**의 조합으로 구성하여 흐름을 직관적으로 시각화하고, 현대적이고 세련된 UX를 제공.

---

## 2. 수평 정사각형 결재선 UI 설계안
결재선 빌더 및 문서 상세 뷰에서 결재권자 정보는 다음과 같이 정사각형 형태의 카드가 좌에서 우로 나란히 늘어서고 사이사이에 화살표가 이어지는 구조로 디자인합니다.

```
+-----------+        +-----------+        +-----------+        +-----------+
|   기안자  |        |  1차 결재 |        |  2차 결재 |        |   최종    |
|  [홍길동] |  --->  |  [이몽룡] |  --->  |  [성춘향] |  --->  |  [완료]   |
|   (상신)  |        |   (승인)  |        |   (대기)  |        |   (전송)  |
+-----------+        +-----------+        +-----------+        +-----------+
```

### 카드 구성 요소
1. **헤더 영역:** 결재 순번 또는 역할 구분 (예: `기안`, `결재`, `전결`, `합의`)
2. **바디 영역:** 사용자 아바타, 사원명 및 직급, 소속 부서
3. **푸터 영역:** 현재 결재 진행 상태별 색상 뱃지 (예: `대기`는 회색, `진행중`은 오렌지색, `승인/완료`는 하늘색 또는 민트/녹색)
4. **연결 화살표:** 각 카드 우측에 배치하여 다음 단계로 흐르는 순서를 부각.

---

## 3. 참조자(수신/참조) 표현 방안 조사 및 제안
참조자(및 수신처)는 결재선 흐름처럼 **순차적 승인(Sequential Approval)을 하지 않고**, 기안 시점에 공유되거나 결재 완료 후에 문서 조회가 가능해지는 **비동기 열람 대상**입니다. 따라서 이들을 결재 흐름선 내부에 동일하게 나열하는 것은 결재 체인의 혼선을 줍니다. 

이에 따라 참조자를 표현하기 위한 **3가지 UI 대안**을 비교 분석합니다.

### [대안 A] 최종 완료 노드 하단 브랜치 연결 (Recommended)
최종 승인(`완료`) 카드 밑에 점선 또는 별도 분기 화살표를 내려 **"문서 열람 및 수신 그룹"** 형태로 카드 뭉치나 목록을 묶어 연출하는 기법입니다.
* **장점:** "결재 완료 후 수신 및 참조처로 문서가 최종 도달한다"는 문서 생명주기 흐름에 완벽히 일치하여 사용자가 이해하기 가장 쉽습니다.
* **단점:** 결재 단계가 길어지면 하단 브랜치 레이아웃이 좌측 정렬 등의 변수에 의해 꼬일 수 있어 반응형 CSS 세밀 조정이 필요합니다.

### [대안 B] 결재 흐름 우측 상단 플로팅 보조 패널
메인 결재선 박스 위나 우측 여백에 작은 배지 스타일로 `📨 참조: 개발팀, 이과장...` 형식의 가볍고 미니멀한 칩(Chip) 컴포넌트로 분리 배치합니다.
* **장점:** 메인 결재 카드 흐름을 전혀 간섭하지 않으므로 결재 순서가 명료하게 유지되며 화면 공간을 매우 효율적으로 사용합니다.
* **단점:** 참조자가 누구인지 집중해서 찾지 않으면 눈에 띄지 않을 수 있습니다.

### [대안 C] 기안(시작) 카드 하부에 병렬 점선(Dashed) 레일 배치
결재선 메인 레일은 실선(Solid Line)과 실물 카드로, 바로 아래에 참조 레일을 별도로 생성하여 점선(Dashed Line) 화살표를 길게 뻗어 연결하는 방식입니다.
* **장점:** 병렬 관계에 있는 열람 권한자임을 시각적으로 가장 전문적인 설계 도식(Flowchart) 형태로 명확히 표현할 수 있습니다.
* **단점:** 모바일이나 좁은 화면 영역에서 2단 레일 레이아웃이 겹치거나 깨지는 스크롤 이슈가 발생할 수 있습니다.

## 4. 최종 제안안 (선택 모델 및 CSS 구현 계획)
우리는 **[대안 A]**의 확장형 또는 **[대안 B]**의 직관적 배지 결합 모델을 채택합니다.

```
 [기안 (홍길동)] ➔ [1차 결재 (이몽룡)] ➔ [2차 결재 (성춘향)] ➔ [최종 완료]
                                                                    │
                                                            ┌───────┴───────┐
                                                     (수신: 개발팀)  (참조: 김대리)
```

### 마크업 설계 예시 (React / Tailwind CSS)
* **결재선 컨테이너:** `flex flex-wrap items-center gap-3 md:gap-4`
* **정사각형 카드:** `w-28 h-28 flex flex-col justify-between rounded-xl border border-border bg-panel p-2 text-center shadow-sm`
* **화살표 컴포넌트:** `text-ink3 text-[14px] font-bold self-center`
* **참조자 카드:** 메인 결재선과 구분되는 점선 테두리(`border-dashed border-teal/40 bg-teal-soft/5`)와 둥근 배지 형태의 미니멀 스타일로 흐름선 하단부에 별도 그룹화하여 통일감 부여.

## 5. 단계별 적용 일정
1. **1단계 (기획 및 조사):** 본 보고서를 기반으로 개발 방향성 조율 및 컴포넌트 구조 확정 (완료)
2. **2단계 (카드 및 화살표 CSS 스타일 교체):** `ApprovalLineBuilder.tsx` 및 `ApprovalDocumentView.tsx` 내 결재선 드로잉 로직 개선
3. **3단계 (참조자 칩/브랜치 디자인 고도화):** 수평 라인 하단에 콤팩트한 참조/수신인 카드 흐름 렌더링
4. **4단계 (동작 테스트):** 전결, 합의 등 복합 결재선 시나리오에서도 정사각형 카드가 유연하게 스크롤(Overflow-X) 및 래핑되는지 검증

## 6. 기안(작성) 화면 vs 조회(상세) 화면의 역할 분담 설계

결재권자 지정 및 흐름 정보 시각화의 효율성과 속도를 모두 확보하기 위해 작성 단계와 조회 단계를 이원화하여 역할을 분담합니다.

* **기안 작성 화면 (입력 및 설정 영역): 실용적이고 빠른 폼 형태 유지**
  - **목적:** 기안자가 신속하게 대상자를 선택하고 결재선을 신속히 조율하는 입력 본연의 속도에 초점을 맞춤.
  - **구조:** 기존의 경량화된 목록형/수평 라인 빌더 인터페이스를 유지하여 복잡한 그래픽 로딩이나 렌더링 딜레이를 방지하고 편의성 극대화.
  
* **결재 진행 / 상세 조회 화면 (시각화 영역): 예쁜 플로우차트 형태 적용**
  - **목적:** 문서를 최종 검토하는 결재자나 열람자가 "현재 누구 차례이며 최종 누구에게 도달하는지" 한눈에 확인하도록 시인성 극대화.
  - **구조:** 가독성을 높인 정사각형 카드 + 연결 화살표(➔) + 최종 노드 하부 분기(참조/수신 그룹) 융합 레이아웃을 통해 세련된 디자인 및 만족도 제공.
