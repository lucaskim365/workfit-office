# 🖋️ 전결 규정 기반 통합 결재서식 및 결재선 룰 조건 확장 계획

본 문서는 전결 규정 PDF의 **번호가 매겨진 셀(예: 1) 휴가, 3) 식대 등) 단위로만 결재서식(Form)**을 설계하고, 그 아래의 **세부 구분(가, 나, 다 등)은 서식 내 드롭다운 박스**를 통해 선택하도록 개편하기 위한 가이드라인 및 구현 계획입니다.

이 구조에 맞춰 서식을 선택했을 때 입력 항목이 동적으로 변환되며, 결재선 룰 관리기(Rule Editor)에서도 특정 서식을 고른 후 **드롭다운 항목(가, 나, 다 등)을 조건으로 선택하여 결재선 규칙이 분기**되도록 구현합니다.

## 📅 1. 통합 결재서식 목록 및 조건부 결재선 정의

*   **총무 (General Affairs)**

    *   **해외출장 신청서**: 금액 구간 및 서브 조건별 결재선 분기

    *   **회식비 청구서**: 회식비 구분(가. 부서원 회식 / 나. 특별회식) 드롭다운 분기

    *   **교통비 청구서**: 교통비 구분(야근택시, 외근교통, 외근택시, 주유비) 드롭다운 분기

*   **품의 (Approval/Request)**
    *   **일반 품의서 (자산/비품/일반경비)**: 금액 구간별 결재선 분기
    *   **원자재 품의서**: 금액 구간별 결재선 분기
    *   **외주개발/용역비 품의서**: 고정 결재선 적용
    *   **소모품 신청서**: 금액 구간별 결재선 분기
*   **경조사 (Congratulation & Condolence)**
    *   **경조 지원 신청서**: 지원 구분(가. 거래처화환 / 나. 임직원화환 / 다. 거래처 축부의금) 드롭다운 분기

## ⚙️ 2. 조건부 동적 결재선 및 UI/UX 구현 계획

### 2.1 결재서식 스키마 및 동적 입력 폼 확장
*   **서식 내 드롭다운 변경에 따른 양식 변경**:
    *   서식 상세 필드 설정(`FormField`)에 특정 드롭다운 값에 따라서만 노출될 필드를 지정할 수 있도록 `visibleIf` 또는 관계 스키마를 보강합니다.
    *   예: `보험 관련 신청서`에서 '자동차보험' 선택 시 차량번호 입력 필드 활성화, `운반비 청구서`에서 '우편/택배' 선택 시 발송 물품 명세서 필드 노출 등.

### 2.2 결재선 룰 엔진 스키마 확장 (`ApprovalRouteRule`)
결재선 매칭 엔진이 서식(`docType`)뿐만 아니라, 서식 내 드롭다운 선택 조건(`subCategory`)도 검사할 수 있도록 룰 마스터 스키마를 확장합니다.

```typescript
export const approvalRouteRuleSchema = z.object({
  id: z.string(),
  name: z.string(),
  priority: z.number().int().default(100),
  active: z.boolean().default(true),
  
  // 기존 적용 조건
  docType: z.string(), // 예: '보험', '교통비'
  
  // 신규 적용 조건 추가
  /** 서식 내 드롭다운 매칭 키값 (선택형 필드 key와 매칭, 예: 'insuranceType') */
  conditionKey: z.string().nullable().default(null),
  /** 매칭할 서브 카테고리 값 목록 (가, 나, 다 값 목록, 예: ['화재보험', '단체상해보험 재가입']) */
  conditionValues: z.array(z.string()).default([]),

  deptScope: deptScopeSchema,
  positionFromRank: z.number().nullable(),
  positionToRank: z.number().nullable(),
  amountFrom: z.number().nullable(),
  amountTo: z.number().nullable(),
  steps: z.array(routeStepSchema),
});
```

*   **룰 매칭 엔진 업데이트 (`ruleMatches`)**:
    *   상신서 본문 데이터(`docData`)에서 `conditionKey`에 해당하는 값을 찾아, 규칙에 정의된 `conditionValues` 배열 안에 포함되어 있는지 여부를 추가로 검사하여 룰을 최종 매칭합니다.

### 2.3 결재선 규칙 관리 UI/UX 개편
*   **규칙 편집 화면(Rule Editor)**:
    *   특정 결재서식(`docType`)을 선택하면, 해당 서식이 갖고 있는 선택형(Dropdown) 필드 목록을 동적으로 분석하여 **하위 구분 조건(가. ~ 항목 목록)**을 선택할 수 있는 다중 선택 콤보박스 인터페이스를 제공합니다.
    *   이를 통해 관리자가 "보험 관련 신청서 서식 중, 보험 종류가 '화재보험' 또는 '단체상해보험 재가입'인 경우"에 작동하는 결재선을 룰 편집 화면에서 직접 설정할 수 있습니다.

## 📁 2. 결재서식 폴더(분류) 기능 구현 계획

사용자가 결재서식 목록을 효율적으로 분류하고 관리할 수 있도록 폴더 구조 관리 및 서식 배치 기능을 추가합니다.

### 2.1 데이터 모델 및 스키마 확장

```typescript
/** 1. 폴더(Folder) 도메인 스키마 추가 */
export const approvalFolderSchema = z.object({
  id: z.string(),              // 폴더 식별자 (예: 'dept-hr', 'dept-ga')
  name: z.string(),            // 폴더 이름 (예: '인사', '총무')
  order: z.number().default(0) // 정렬 순서
});
export type ApprovalFolder = z.infer<typeof approvalFolderSchema>;

/** 2. 기존 결재서식 스키마(ApprovalForm) 확장 */
// src/domain/approvalForm/schema.ts 내에 folderId 필드 추가
export const approvalFormSchema = z.object({
  // ... 기존 필드
  folderId: z.string().nullable().default(null) // 소속 폴더 ID (null이면 최상위 루트)
});
```

### 2.2 폴더 관리 UI/UX 및 기능 설계

1.  **폴더 생성 및 드래그 앤 드롭 관리**
    *   서식 목록 헤더 바 우측에 `+ 폴더` 생성 버튼을 추가합니다.
    *   생성된 폴더와 서식 아이템들은 드래그 앤 드롭(Drag & Drop) 라이브러리를 통해 트리 구조로 상호간의 정렬 위치 및 소속(폴더 내 이동)을 실시간 편집할 수 있습니다.
2.  **폴더 수정 및 삭제**
    *   폴더에 마우스 우클릭 시 컨텍스트 메뉴(이름 변경, 삭제)가 표시됩니다.
    *   **폴더 삭제 시 정책**: 폴더를 삭제하면 폴더 자체는 제거되지만, **폴더에 소속되어 있던 문서 서식들은 삭제되지 않고 최상위 루트(`folderId = null`)로 안전하게 대피**시킵니다.
3.  **서식 편집 화면 연동**
    *   서식을 생성하거나 편집할 때, 현재 생성된 폴더 목록을 셀렉트 박스(Dropdown) 형태로 노출하여 속할 폴더를 지정하거나 변경할 수 있도록 구성합니다.

### 2.3 기안 작성 화면(Modal) 분류 적용

*   새 기안 작성 시 단순히 서식을 나열하는 것이 아니라, 사용자가 메신저 조직도를 탐색하듯 **폴더 구조를 클릭하여 폴더별 소속 서식만 깔끔하게 조회**할 수 있는 아코디언/트리 뷰를 제공하여 작성 화면의 탐색 피로도를 크게 줄입니다.
