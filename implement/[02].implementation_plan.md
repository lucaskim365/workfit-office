# [2-1단계] 메신저 기능 고도화 개발계획서

본 계획서는 메신저 시스템의 사용성을 고도화하기 위해 ① 조직도 트리뷰 연동을 통한 멤버 선택, ② 부서 개설 시 부서원 단체 채팅방 자동 개설(Mock), ③ 이전 대화 검색 필터링 및 히스토리 풀 조회 기능을 구현하는 방안을 다룹니다.

---

## 🛠️ 주요 변경 사항 및 기술 설계

### 1. 순수 조직도 트리 기반의 멤버 선택기 (MemberPicker)
- **위치**: [QuickDock.tsx](file:///c:/WorkFit/전자결재시스템/workfit-office/src/app/shell/QuickDock.tsx) 내 `MemberPicker`
- **변경 사항**:
  - 기존의 텍스트 검색 입력 및 리스트 탭 토글 방식을 **완전히 제거**합니다.
  - 화면 진입 시 오직 **부서 계층 구조(Tree View)와 각 부서별 소속 임직원**이 트리 형태로만 렌더링되도록 구성합니다.
  - 각 부서 노드는 접고 펼칠 수 있는 토글(▶/▼)을 기본 탑재하고, **사용자 이름을 클릭**하여 간편하게 선택할 수 있게 합니다.

---

### 2. 현재 부서 기반의 동적 부서 단톡방 자동 매핑
- **동적 단톡방 생성 메커니즘**:
  - [chatRoom.repo.ts](file:///c:/WorkFit/전자결재시스템/workfit-office/src/data/chatRoom/chatRoom.repo.ts)의 `list` 메서드 실행 시, 현재 시스템에 등록된 전체 부서 정보(`departments`)와 사용자 정보(`users`)를 취합합니다.
  - 각 부서마다 고유 ID(`RM-DEPT-[부서ID]`)를 가지는 `[부서명] 단체방`을 자동으로 생성하여 데이터베이스(Firestore/Memory)에 반영합니다.
  - 해당 단톡방의 `members` 배열에는 오직 **그 부서에 속한 활성 사용자(status: '사용')들의 ID**만 동적으로 채워집니다.
- **참여 권한 격리**:
  - 메신저 목록 조회 시 본인의 사용자 ID가 `members` 배열에 들어있는 방만 필터링되어 출력되므로, **오직 본인이 속한 부서의 단체 채팅방만 노출 및 참여**할 수 있게 됩니다.
  - 기존의 하드코딩되었던 무작위 단톡방 씨드(생산1팀 단톡방 등)는 씨드 목록 및 DB에서 안전하게 제외합니다.

---

### 3. 메시지 검색 및 전체 히스토리 조회
- **대화 내 검색**:
  - 대화방 내부 상단에 대화 검색 기능(🔍)을 탑재하여 텍스트 및 첨부파일명에 대해 실시간 필터링을 제공합니다.
- **히스토리 풀 조회**:
  - 대화방 스크롤을 끝까지 올리거나 스크롤 시 누적된 대화 히스토리가 끊김 없이 역순으로 원활히 출력되도록 Repository 구조를 확인 및 최적화합니다.

## 🧪 Verification Plan

### Manual Verification
1. **순수 조직도 트리 선택기 확인**:
   - 메신저의 '새 대화' 또는 '멤버 초대' 클릭 시 검색 탭 없이 **오직 조직도 트리뷰만 렌더링**되는지 확인.
   - 부서 노드를 펼쳐 임직원을 선택하고 대화방이 잘 생성되는지 검증.
2. **부서별 단톡방 격리 및 동적 생성 확인**:
   - 홍채원 사원(`S/W 개발팀`)으로 로그인 시, 메신저 목록에 `S/W 개발팀 단체방`만 보이고 타 부서 단톡방(예: 품질관리팀 단체방)은 전혀 보이지 않는지 검증.
   - 임의의 다른 부서 사용자(예: 품질관리팀 강윤석)로 로그인 시, 오직 본인 부서인 `품질관리팀 단체방`만 참여 리스트에 뜨는지 교차 검증.
3. **히스토리 검색 기능**:
   - 대화 내에서 특정 키워드를 검색했을 때 실시간으로 해당하는 메시지만 필터링되어 노출되는지 검증.
