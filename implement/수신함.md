# ✉️ 전결 규정 '시행자' 자동 라우팅 구현 및 수신함 확장 기획안

본 문서는 전결 규정 PDF의 **비고-시행** 열에 근거한 특정 결재 완료 문서의 기안자 수신함 자동 라우팅 구현 상세와, 향후 동적/고정형 수신처 확장을 위한 기획 방안을 정리한 문서입니다.

---

## 1. ⚙️ '시행: 기안자' 결재 완료 문서 자동 수신 라우팅

전결 규정상 최종 승인이 종결된 후 기안자가 해당 업무를 직접 실행(시행)해야 하는 서식들에 대해, 결재 완료 시 기안자의 **수신함**에 문서가 자동으로 노출되도록 하는 연동 로직을 구현했습니다.

### 1.1 대상 결재서식 종류
* **인사 (HR)**: `외근 신청서`
* **총무 (GA)**: `국내출장 신청서`, `해외출장 신청서`, `인장 날인 요청서`, `공문 발송 신청서`

### 1.2 구현 상세
* **적용 파일**: [src/domain/approvalDoc/engine.ts](file:///c:/WorkFit/전자결재시스템/workfit-office/src/domain/approvalDoc/engine.ts)
* **적용 방식**: 결재함 목록 필터 함수인 `matchesBox`의 `box === '수신'` 로직 내에 아래의 조건을 추가로 판정하여 노출되도록 반영했습니다.
  * 최종 완료된 문서(`doc.status === '완료'`)
  * 기안자 본인이 로그인한 사용자(`doc.drafterId === userId`)
  * 문서 종류(`doc.docType`)가 위 5개 서식에 해당

```typescript
if (box === '수신') {
  const isExecutorDrafter = doc.drafterId === userId && ['외근', '국내출장', '해외출장', '인장날인', '공문발송'].includes(doc.docType);
  return isExecutorDrafter || doc.steps.some((s) => s.kind === '참조' && s.approverId === userId);
}
```

## 📅 2. 향후 수신처(시행처) 지정을 위한 확장 기획안 (3가지)

추후 시스템 고도화 시 특정 부서 또는 조직원의 수신함으로 문서를 정적/동적으로 유연하게 보낼 수 있도록 하기 위한 3가지 접근 방식입니다.

### 💡 기획 1: 결재서식별 고정 수신처 지정 방식 (서식 마스터 연동)
* **개요**: 결재서식 스키마(`ApprovalForm`)에 `recipientDeptId` 또는 `recipientUserId` 필드를 내장하여, 서식 마스터 레벨에서 수신 부서나 인원을 정적으로 연결하는 구조입니다.
* **시나리오**:
  * 관리자가 서식 설정에서 `소모품 신청서` 완료 시 수신처로 `구매팀`을 매핑해 둡니다.
  * 기안자가 문서를 작성하고 결재가 최종 완료되면, 별도의 설정 없이 자동으로 구매팀 소속원 전체의 수신함으로 문서 사본이 배달됩니다.
* **장점**: 서식별로 처리 담당 부서가 일정하고 고정되어 있는 일반적인 사내 행정 업무에 가장 직관적이고 설정이 간단합니다.

### 💡 기획 2: 결재선 규칙(`ApprovalRouteRule`) 기반 조건부 수신 분기 방식 (룰 엔진 연동)
* **개요**: 결재선 룰 스키마에 결재선(steps) 외에 **수신처 규칙(recipients)**을 신설하여, 룰 엔진이 조건에 맞춰 수신 대상을 동적으로 분석 및 발송하는 방식입니다.
* **시나리오**:
  * `보험 관련 신청서` 룰에서 세부 드롭다운 구분이 `자동차보험`일 때만 수신처를 `총무팀`으로 동적 지정합니다.
  * `화재보험`일 때는 수신처를 `재경팀` 및 `대표이사` 수신함으로 다중 라우팅합니다.
* **장점**: 동일한 서식이더라도 작성 조건이나 금액, 기안자 부서에 따라 문서를 받아볼 실행(시행) 부서가 달라져야 하는 복잡한 전결 요건을 완벽히 흡수할 수 있습니다.

### 💡 기획 3: 기안 작성 시 사용자 수동 지정 방식 (상신 화면 피커 제공)
* **개요**: 기안서 작성 창(`ApprovalDraftModal`) 내 결재선 빌더 영역 하단에 '수신(시행)처 지정' 기능을 신설하여, 기안자가 상신서를 올릴 때 직접 검색해 수신 부서나 인원을 리스트에 담아 기안하는 방식입니다.
* **시나리오**:
  * 기안자가 `품의서` 상신 시 타 부서의 업무 협조가 수반됨을 인지하고, 수신 부서 피커를 열어 관련 부서인 `AX사업본부`를 수동 지정하여 결재를 요청합니다.
* **장점**: 정해진 룰 외에 기안 건마다 협조/수신 부서가 매번 유동적으로 바뀔 수 있는 업무 성격에 가장 유연하고 민첩하게 대처할 수 있습니다.
